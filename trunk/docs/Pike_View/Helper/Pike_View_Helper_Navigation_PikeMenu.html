With PiKe menu you can render a menu with an addidtional permission check. The normal navigation

menu helper has support for setting only one role with setRole. Some applications users are allowed to have more roles and when menu is rendered it needs to check the user may see an entry based on __multiple__ roles! Check it out!

It work's very simple, in your frontcontroller plugin or bootstrap where you set up your website navigation replace your current menu implementation by the following:

  1.  $config = new Zend_Config_Xml($fileToANavigationXML, 'nav');
  2.  $navigation = new Zend_Navigation($config);
  3.  
  4.  $arrayOfroles = Zend_Auth::getInstance()->getIdentity()->getRoles();
  5.  
  6.  $layoutView->navigation()
  7.       ->PikeMenu($navigation)
  8.       ->setACL(Zend_Registry::get('acl')) //should be a object of type Zend_ACL
  9.        ->setRoles($arrayOfRoles);

And make sure u use PikeMenu either in your view (layout script proberly)

<?= $this->navigation()->PikeMenu(); ?>

Next make sure the Pike library is loaded with autoloaderNamespaces[] = "Pike" in your application.ini. And beyond to make sure the view navigation helper is found add something like the following to it:

resources.view.helperPath.Pike_View_Helper_Navigation_ = APPLICATION_PATH "/../library/Pike/View/Helper/Navigation"
